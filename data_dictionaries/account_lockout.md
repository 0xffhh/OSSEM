# Account Lockout Data Dictionary

Security events that are generated by a failed attempt to log on to an account that is locked out.

## Data sources schema applies to

| OS | Data source | Event ID | Description |
|--------|---------|-------|---------|
| Windows | Security | 4625 | An account failed to log on |

## References

* https://github.com/MicrosoftDocs/windows-itpro-docs/blob/master/windows/security/threat-protection/auditing/event-4625.md

## Data Fields

| field name | field type | description | valid values |
|--------|---------|-------|---------|
| reporter_user_sid | string | SID of account that reported information about logon failure | S-1-5-18  |
| reporter_user_name | string | The name of the account that reported information about logon failure | DC01$ |
| reporter_user_domain | string | The name of the domain of the user that reported the logon failure | rivendell |
| logon_type | integer | the type of logon which was performed | 3 (Network) |
| user_sid | string | SID of the account that was specified in the logon attempt | S-1-0-0  |
| user_name | string | Name of the account that was specified in the logon attempt | wardog |
| user_domain | string | Information about the domain name of the account being used for the logon attempt. Usually here you see domain or computer name | rivendell |
| logon_failure_reason | string | textual explanation of Status field value. For this event it typically has “Account locked out” value | Account locked out |
| logon_failue_status | string | the reason why logon failed. For this event it typically has “0xC0000234” value.| 0xC0000234 |
| logon_failure_sub_status | string | additional information about logon failure | 0XC000006D |
| process_id | integer | Process ID of the process that attempted the logon | 216 |
| process_name | string | The name of the process that attempted the logon | winlogon.exe |
| process_path | string | The complete path and name of the executable associated with the process that attempted the logon | C:\Windows\System32\winlogon.exe |
| host_name | string | machine name from which logon attempt was performed | WK001 |
| src_ip | string | IP address of machine from which logon attempt was performed | 127.0.0.1 |
| src_port_number | integer | source port which was used for logon attempt from remote machine | 0 |
| logon_process_name | string | the name of the trusted logon process that was used for the logon attempt | User32 |
| logon_authentication_package | string | The name of the authentication package which was used for the logon authentication process. | NTLM |
| logon_transmitted_services| string | [Kerberos-only]: the list of transmitted services. Transmitted services are populated if the logon was a result of a S4U (Service For User) logon process | - |
| logon_package_name | string | The name of the LAN Manager sub-package (NTLM-family protocol name) that was used during the logon attempt | NTLM V2 |
| logon_key_length | integer | the length of NTLM Session Security key. Typically it has 128 bit or 56 bit length. This parameter is always 0 if “Authentication Package” = “Kerberos”, because it is not applicable for Kerberos protocol. This field will also have “0” value if Kerberos was negotiated using Negotiate authentication package | 0 |
